Django 架構理論
Django 的 MVT 是什麼？
Model就是用來跟資料庫溝通的地方，裡面定義資料長什麼樣子，像是欄位有哪些、資料型態是什麼，還有資料怎麼存取、更新、刪除都在這邊處理。簡單說，就是負責管理資料的結構和操作。
View 處理使用者的請求，負責邏輯，從 Model 取得資料，然後傳給 Template
Template 負責前端呈現，把資料轉成 HTML 顯示給使用者
流程:使用者送出請求 → View 接收請求並處理 → 使用 Model 抓資料 → 傳資料給 Template 顯示畫面

django中的urls
urls.py 就像網站的導航地圖，負責把使用者輸入的網址，對應到我們寫好的 view 函式。這些 view 函式會處理後端的邏輯，像是從資料庫取資料或處理使用者輸入，最後再把結果傳給 template 呈現成網頁給使用者看。簡單說，urls.py 是讓網站知道不同網址要用哪個功能，然後怎麼呈現畫面。

ORM 是什麼？
ORM 用 Python的class來操作資料庫，不需要寫 SQL 語法。它比較安全也更容易管理資料結構,只要寫 .create()、.filter() 這種語法，就可以新增或查詢資料了
